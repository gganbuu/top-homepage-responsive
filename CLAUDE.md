# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

A static single-page portfolio/homepage site (hero section, project cards, contact footer), bundled with webpack. There is no JS framework and no application logic — `src/index.js` only imports the stylesheet; all content lives in `src/template.html` and all layout/visuals in `src/styles.css`.

## Commands

- `npm start` — runs `webpack serve` (dev server with live reload; watches `src/template.html` explicitly, per `webpack.config.js`).
- There is no build script wired up in `package.json` beyond `start`; use `npx webpack` directly for a production-mode build if needed (config `mode` is currently hardcoded to `"development"`).
- No test suite exists — `npm test` is a stub that exits with an error.
- No linter is configured.

## Architecture

- `src/template.html` — the entire page markup (hero/about-me, projects grid, contact footer). Injected via `HtmlWebpackPlugin` using this file as the template.
- `src/styles.css` — all styling, imported directly in `src/index.js` via `import "./styles.css"` and processed by `style-loader`/`css-loader`.
- `src/index.js` — webpack entry point; currently has no purpose other than pulling in the CSS.
- `src/fonts/`, `src/img/`, `src/svg/` — static assets referenced by relative path from HTML/CSS; images/fonts are emitted via webpack's `asset/resource` rules, HTML is processed by `html-loader`.
- `design_spec/` — reference mockups (desktop/tablet/mobile PNGs) that the markup/CSS are meant to visually match. Consult these when making layout changes.
- `dist/` — build output, not source; regenerated by webpack (`clean: true`).

## Content notes

- The projects grid (`.project-card` entries) and hero/contact text are currently placeholder Lorem ipsum / "Project name" content — real content has not yet been filled in.
- Project card accent colors are set via IDs (`#red`, `#purple`, `#green`, `#pink`, `#orange`, `#navy`) on `.project-image`, not classes — follow this pattern if adding more cards.
- The page relies on an external CDN stylesheet (devicon) linked directly in `template.html`'s `<head>`.
